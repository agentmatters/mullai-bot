# Mullai Observability Stack

This folder contains the Docker Compose configuration to run a local observability stack for the Mullai application. The stack is designed to collect, process, and visualize traces and metrics generated by the Mullai Host using OpenTelemetry.

## Components

The stack consists of the following services:

1. **OpenTelemetry Collector (`otel-collector`)**: The central hub that receives telemetry data (traces, metrics) from the Mullai application via the OTLP protocol. It processes this data and exports it to the appropriate backends.
2. **Jaeger (`jaeger`)**: A distributed tracing system used to visualize the traces sent from the application.
3. **Prometheus (`prometheus`)**: A time-series database used to store and query metrics.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) and Docker Compose installed on your local machine.

## Getting Started

### Starting the Stack

To start the observability stack in the background, navigate to this directory and run:

```bash
docker-compose up -d
```

### Stopping the Stack

To stop and remove the containers, run:

```bash
docker-compose down
```

## Accessing the UIs

Once the stack is running, you can access the respective dashboards using your web browser:

- **Jaeger UI (Traces)**: [http://localhost:16686](http://localhost:16686)
- **Prometheus UI (Metrics)**: [http://localhost:9090](http://localhost:9090)

## Application Configuration

The `Mullai.Host` application is already configured to automatically send OpenTelemetry data to this local stack by default when running locally. 

If you need to explicitly define or override the endpoint (for example, if running the app in a different network configuration), you can set the following environment variable before starting the `Mullai.Host`:

```bash
export OTEL_EXPORTER_OTLP_ENDPOINT="http://localhost:4317"
```

## Exposed Ports Reference

- **`4317`**: OTLP gRPC payload receiver (Mullai application sends data here)
- **`4318`**: OTLP HTTP receiver
- **`8889`**: Prometheus metrics endpoint (Collector exposes metrics here for Prometheus to scrape)
- **`16686`**: Jaeger Web UI
- **`9090`**: Prometheus Web UI
